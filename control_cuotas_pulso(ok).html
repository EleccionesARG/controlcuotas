Cómo usar el sistema
Flujo de trabajo recomendado
1
Cargá la tabla de referencia de estratos (una sola vez). Andá a Cargar datos → ① Tabla de estratos. Exportá tu Google Sheet de referencia como CSV: Archivo → Descargar → Valores separados por comas. El CSV necesita 3 columnas: provincia, departamento, estrato.
2
Configurá las cuotas en la pestaña Cuotas. Definí los grupos de edad, N total y el objetivo por cada celda cruzada. Los estratos se detectan automáticamente de la tabla de referencia.
3
Cargá los datos de la encuesta: Cargar datos → ② Datos de encuesta. Exportá el sheet de SurveyMonkey como CSV o pegá los datos directamente. Mapeá las columnas de género, edad y provincia. Para los departamentos: SurveyMonkey genera una columna por provincia — indicá qué columna corresponde a cada una.
4
Actualizaciones periódicas: para actualizar los datos, simplemente re-exportá el CSV de SurveyMonkey y volvé a cargarlo. Podés reemplazar todos los datos o acumularlos. El sistema acepta 2500–3000 filas sin problema.
Publicar en línea con Firebase + GitHub Pages

Seguí estos pasos para que tus socios vean el dashboard en tiempo real desde cualquier navegador. Tiempo estimado: 10–15 minutos.

A
Crear proyecto en Firebase
Entrá a console.firebase.google.com con tu cuenta de Google. Hacé clic en "Crear un proyecto", ponerle nombre (ej: pulso-cuotas), desactivar Google Analytics si querés, y finalizar.
B
Activar Realtime Database
En el menú izquierdo: Compilación → Realtime Database → Crear base de datos. Elegí ubicación us-central1 (o cualquiera). Cuando pregunte las reglas, elegí "Empezar en modo de prueba" y hacé clic en Habilitar.
C
Cambiar las reglas de seguridad
En Realtime Database, hacé clic en la pestaña "Reglas". Reemplazá todo el contenido con esto y publicá:
{ "rules": { ".read": true, ".write": true } }
Esto permite que cualquiera con el link pueda leer y vos escribir. Podés restringirlo después con autenticación.
D
Copiar la configuración
Hacé clic en el ⚙️ (arriba a la izquierda) → Configuración del proyecto. Bajá a "Tus apps", hacé clic en el ícono </> (Web), registrá la app con cualquier nombre. Te va a mostrar un bloque const firebaseConfig = {...} con 7 valores. Copiá esos 7 valores.
E
Pegarlo en el archivo HTML
Abrí el archivo control_cuotas_pulso.html con el Bloc de notas (clic derecho → Abrir con → Bloc de notas). Buscá (Ctrl+F) el texto PASTE_API_KEY y reemplazá las 7 líneas con los valores que copiaste de Firebase. Guardá el archivo.
F
Subir a GitHub Pages
Entrá a github.com y creá una cuenta si no tenés. Creá un repositorio nuevo: New repository → nombre (ej: pulso-cuotas) → Público → Create.
Subí el archivo: Add file → Upload files, arrastrá el HTML, hacé commit.
Activá GitHub Pages: Settings → Pages → Branch: main → / (root) → Save.
En unos segundos aparece el link: https://tuusuario.github.io/pulso-cuotas/control_cuotas_pulso.html
G
Verificar que funciona
Abrí el link. En el sidebar deberías ver ● Conectado a Firebase. Cargá un caso desde tu computadora y abrí el mismo link en otro navegador o dispositivo — el cambio debería aparecer en segundos.
Fórmulas equivalentes en Google Sheets

Si querés replicar la lógica en tu propio sheet:

Asignar estrato (ÍNDICE + COINCIDIR)
=IFERROR(INDEX(Ref!$C:$C,MATCH(D2&"|"&E2,Ref!$A:$A&"|"&Ref!$B:$B,0)),"Sin estrato")

D2 = provincia, E2 = departamento. Col C en hoja Ref = estrato.

Agrupar edad automáticamente
=IFS(C2<18,"Menor",C2<=29,"18-29",C2<=44,"30-44",C2<=59,"45-59",C2>=60,"60+")
Contar casos por celda cruzada
=COUNTIFS(Datos!$B:$B,A2,Datos!$D:$D,B2,Datos!$F:$F,C2)

B=Género, D=GrupoEdad, F=Estrato. A2/B2/C2 son los valores objetivo en la hoja Cuotas.

Semáforo de estado
=IFS(D2=0,"⬜ Sin iniciar",D2/E2<0.8,"🟢 En curso",D2/E2<1,"🟡 Por cerrar",D2>=E2,"🔴 Completa")
